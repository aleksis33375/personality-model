## Дизайн

**Цветовая палитра — не менять:**
```
--coral: #D97070
--green: #57B374
--blue:  #6688CC
--gold:  #FFD700
--dark:  #080810
--dark2: #0e0e1a
--text:  #e8e8f0
--muted: #8888aa
```

**Принципы:**
- Тёмная тема, фон `#080810`
- Градиентный текст: `linear-gradient(90deg, --coral, --gold, --green)` через `-webkit-background-clip: text`
- Фоновые блобы: `filter: blur(90px)`, `opacity: 0.13`, анимация `blobDrift`
- Появление элементов: анимация `fadeUp` (opacity 0→1, translateY 24px→0)
- CTA-кнопка: `linear-gradient(135deg, --coral, --blue)` с анимацией `pulse-cta`
- Карточки: тонкая граница `rgba(255,255,255,0.07)`, hover — цветная тень по теме карточки
- Квиз: прогресс-бар с градиентом, 4 точки-индикатора (coral/green/blue/gold)

**Стек:** чистый HTML + CSS + JS, без фреймворков. `html2canvas` для сохранения результата.

## Адаптивная вёрстка

Все страницы должны хорошо выглядеть на мобильных. Обязательно:
- `clamp()` для размеров шрифтов: `clamp(мин, vw, макс)`
- `max-width` на обёртках, `padding: 0 1.5rem` по бокам
- `grid-template-columns: repeat(auto-fit, minmax(..., 1fr))` для сеток
- Медиазапрос `@media (max-width: 600px)` — уменьшать отступы, переводить в одну колонку
- Чипы приоритета: `flex: 1 1 calc(50% - gap)` чтобы укладывались по два в строке

## Стиль текста

- Язык: русский, обращение на «ты»
- Тон: живой, прямой, без корпоративщины
- Заголовки: короткие, с переносом `<br>` на смысловой паузе
- Подсказки/хинты (`step-hint`): мягкие, объясняют зачем, не давят
- CTA: глагол действия от первого лица или прямой призыв — «Найти свою точку силы», «Написать Алексею»
- Не использовать: «уникальный», «эксклюзивный», «лучший», канцелярит

## Правила работы

**Перед созданием нового файла** — сначала описать: что это за файл, зачем нужен, что в нём будет. Создавать только после подтверждения.

**Перед редактированием** — прочитать файл целиком (или нужную секцию), понять структуру, потом вносить изменения.

**Коммиты** — только по явной просьбе. Сообщения на русском, коротко и по делу.

**Не добавлять без запроса:** комментарии к коду, документацию внутри HTML, лишние классы, обработку ошибок для невозможных сценариев.
